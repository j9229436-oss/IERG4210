## 🎯 Recommended Study Sequence

### **Day 1: Foundation (3-4 hours)**

#### Step 1: Project Overview (15 min)
- Open [CODE_UNDERSTANDING_GUIDE.md](CODE_UNDERSTANDING_GUIDE.md)
- Read only first 3 sections:
  - 📋 Project Overview
  - 🏗️ System Architecture
  - 📁 Core Files (skim)
- **Goal**: Understand "what this does"

#### Step 2: Architecture Deep Dive (1.5 hours)
- Read [CODE_UNDERSTANDING_GUIDE.md](CODE_UNDERSTANDING_GUIDE.md) detailed sections
- Focus on:
  - server.js - Backend app
  - db.js - Database
  - index-dynamic.js - Homepage logic
  - cart.js - Shopping cart
  - admin.js - Admin logic
- **Goal**: Explain what each file does

#### Step 3: Master API Endpoints (45 min)
- Read "API Endpoints" section in [CODE_UNDERSTANDING_GUIDE.md](CODE_UNDERSTANDING_GUIDE.md)
- Write all endpoints on paper
- **Goal**: Remember main routes

#### Step 4: Understand Data Flow (45 min)
- Read "Data Flow Examples" section
- Study 3 scenarios:
  1. User browsing products
  2. User adding to cart
  3. Admin editing product
- **Goal**: Describe complete workflow verbally

---

### **Day 2: Depth (2-3 hours)**

#### Step 5: Learn "Why" Reasoning (2 hours)
- Read [ARCHITECTURE_DECISIONS.md](ARCHITECTURE_DECISIONS.md)
- Focus on decisions:
  - Decisions 1-5: Tech choices
  - Decisions 6-10: Security design
  - Decisions 11-15: Frontend architecture
- **Goal**: Explain why that technology chosen

#### Step 6: Self-Test Basics (30 min)
- Read Level 1 questions in [QA_PRACTICE_QUESTIONS.md](QA_PRACTICE_QUESTIONS.md)
- Answer 5 questions without looking
- Check answers
- **Goal**: Basic questions all correct

---

### **Day 3: Practice (2 hours)**

#### Step 7: Intermediate Challenge (1.5 hours)
- Read Level 2-3 questions in [QA_PRACTICE_QUESTIONS.md](QA_PRACTICE_QUESTIONS.md)
- Try answering 5-6 questions
- Compare with answers
- **Goal**: Understand design principles

#### Step 8: Troubleshooting Prep (30 min)
- Read Level 4 questions
- **Goal**: Know debugging approach

---

### **Final Day: Last Prep (45 min)**

#### Step 9: Fast Review (30 min)
- Read entire [QUICK_REFERENCE.md](QUICK_REFERENCE.md)
- **Goal**: Memorize key numbers and concepts

#### Step 10: Mental Prep (15 min)
- Read [EXAM_DAY_CHEATSHEET.md](EXAM_DAY_CHEATSHEET.md)
- **Goal**: Avoid common mistakes, build confidence

## 🎤 Mock Interview Practice

### Recommended: Self Mock Interview

**Steps**:
1. **Find quiet space**, can record audio
2. **Select 5-10 questions** from [QA_PRACTICE_QUESTIONS.md](QA_PRACTICE_QUESTIONS.md)
3. **Time yourself**: 1-2 min per question
4. **Record your answer**
5. **Listen and evaluate**:
   - Too vague?
   - Too long?
   - Technical errors?
6. **Retry** - practice same questions day 2

**Progression**:
- Practice 1: Might stumble, unclear
- Practice 2-3: Getting fluent
- Practice 4-5: Answer confidently

## 💡 Learning Techniques

### Technique 1: Speak Out Loud
- Don't read silently → verbally explain
- Catches vague understanding immediately

### Technique 2: Draw Diagrams
- Sketch frontend-backend-database flow
- Label API calls and data paths

### Technique 3: Teach Others
- Explain to friend/classmate
- Can't explain clearly = don't fully understand

### Technique 4: Write Notes
- Handwrite API endpoints
- Handwriting cements memory

### Technique 5: Repeat Reviews
- Day 1: Read once
- Day 3: Read again
- Day 5: Quick scan
- Pre-exam: Final review

## ⚠️ Common Study Mistakes

### ❌ Mistake 1: Only Read, Don't Practice
"I read all docs, should be ready"
→ Reading ≠ Can Answer

**Fix**: Practice Level 1-2 questions

---

### ❌ Mistake 2: Memorize, Don't Understand
"I'll just memorize the API list"
→ Interviewer will ask "Why?" - you'll be stuck

**Fix**: Understand "why" reasoning

---

### ❌ Mistake 3: Ignore "Why" Reasoning
"I know what code does"
→ Missing depth that impresses

**Fix**: Read ARCHITECTURE_DECISIONS thoroughly

---

### ❌ Mistake 4: No Mock Practice
"I'm confident, no need to practice"
→ Actual interviews reveal gaps

**Fix**: Do 3-5 mock interviews
